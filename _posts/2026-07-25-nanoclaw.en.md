NanoClaw is a lightweight, container-isolated AI agent that connects OpenClaw to WhatsApp, Telegram, Discord, and other messaging platforms. Built on the OpenClaw Gateway, it provides sandboxed skill execution and agent swarm capabilities.

## 📦 What is NanoClaw?

NanoClaw is a minimal (~15 files) Node.js gateway that:
- **Bridges messaging platforms** to OpenClaw's skill system
- **Runs each skill in a container** for isolation
- **Supports agent swarms** for parallel task execution
- **Manages sessions** across multiple channels

```
NanoClaw Gateway
├── WhatsApp / Telegram / Discord
├── Container Runtime (Skill A, B, C)
└── OpenClaw Core (Models, Memory, Tools)
```

## 🔑 Key Features

### 1. Multi-Channel Support
| Platform | Status | Features |
|----------|--------|----------|
| WhatsApp | ✅ Stable | Text, images, voice, documents |
| Telegram | ✅ Stable | Bot API, inline queries, groups |
| Discord | ✅ Stable | Servers, DMs, threads, reactions |
| Signal | 🔶 Beta | E2E encrypted messaging |
| Slack | 🔶 Beta | Workspaces, channels, threads |

### 2. Container Isolation
Every skill runs in its own Docker container:
```yaml
services:
  skill-excel:
    image: nanoclaw/skill-excel:latest
    isolation:
      network: none
      filesystem: read-only
      memory: 256MB
      cpu: 0.5
```

### 3. Agent Swarms
Deploy multiple AI agents working in parallel:
```json
{
  "swarm": {
    "agents": 5,
    "strategy": "work-stealing",
    "shared_memory": true,
    "max_parallel": 3
  }
}
```

## 🚀 Quick Start

```bash
# Install NanoClaw
npm install -g nanoclaw

# Initialize with messaging channels
nanoclaw init --channels whatsapp,telegram,discord

# Start the gateway
nanoclaw start --port 18791
```

### Docker Compose
```yaml
version: '3.8'
services:
  nanoclaw:
    image: nanoclaw/gateway:latest
    ports:
      - "18791:18791"
    volumes:
      - ./config:/app/config
      - /var/run/docker.sock:/var/run/docker.sock
    environment:
      - OPENCLAW_URL=http://openclaw:18789
```
