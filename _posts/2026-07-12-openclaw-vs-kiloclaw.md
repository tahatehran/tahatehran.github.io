در دنیای دستیارهای هوشمند، دو نام **OpenClaw** و **KiloClaw** زیاد شنیده می‌شوند. در این مقاله، هر دو را بررسی می‌کنیم، معماری زیربنایی OpenClaw را کالبدشکافی می‌کنیم، و تفاوت‌های کلیدی را برای انتخاب بهتر شما آشکار می‌کنیم.

## 🏗️ زیربنای OpenClaw (Architecture)

OpenClaw بر پایه یک **Gateway** (دروازه) ساخته شده است که مسئول مدیریت جلسات، احراز هویت، و اجرای ابزارهاست.

```
┌─────────────────────────────────────┐
│         Gateway (دروازه)            │
│  ┌──────────┬──────────┬──────────┐ │
│  │ Sessions │ Auth     │ Tools    │ │
│  └──────────┴──────────┴──────────┘ │
├─────────────────────────────────────┤
│         Skills & Plugins            │
│  ┌──────────┬──────────┬──────────┐ │
│  │ Office   │ Visual   │ Study    │ │
│  └──────────┴──────────┴──────────┘ │
├─────────────────────────────────────┤
│         Model Providers             │
│  ┌──────────┬──────────┬──────────┐ │
│  │ OpenAI  │ Claude   │ Ollama   │ │
│  └──────────┴──────────┴──────────┘ │
└─────────────────────────────────────┘
```

### لایه‌های اصلی:
1. **Gateway:** مدیریت ارتباطات و احراز هویت
2. **Runtime:** اجرای مدل و پردازش ابزارها
3. **Skills:** قابلیت‌های تخصصی قابل توسعه
4. **Providers:** اتصال به مدل‌های مختلف

## 🔄 OpenClaw در مقابل KiloClaw

| ویژگی | OpenClaw | KiloClaw |
|--------|----------|----------|
| **مجوز** | متن‌باز (Open Source) | تجاری / میزبانی‌شده |
| **استقرار** | خودمیزبان یا ابری | معمولاً ابری |
| **سفارشی‌سازی** | کامل (کد منبع در دسترس) | محدود به تنظیمات |
| **هزینه** | رایگان + هزینه مدل | اشتراک ماهانه |
| **دسترسی به فایل** | کامل (محلی) | محدود به محیط میزبان |
| **امنیت** | دست شما (خودمیزبان) | مدیریت‌شده توسط فراهم‌کننده |
| **اسکیل‌ها** | قابل توسعه نامحدود | بسته به نسخه |

### کی از OpenClaw استفاده کنیم؟
- می‌خواهید کد منبع را کنترل کنید
- نیاز به دسترسی مستقیم به فایل‌های محلی دارید
- حریم خصوصی و امنیت داده اولویت است
- می‌خواهید اسکیل‌های اختصاصی بسازید

### کی از KiloClaw استفاده کنیم؟
- می‌خواهید بدون دردسر فنی شروع کنید
- ترجیح می‌دهید مدیریت زیرساخت با فراهم‌کننده باشد
- به پشتیبانی تجاری نیاز دارید
- تیم کوچکی دارید که نمی‌خواهد سرور نگهداری کند

## 🔐 امنیت در معماری OpenClaw

OpenClaw چندین لایه امنیتی دارد:

1. **HMAC Authentication:** توکن‌های احراز هویت مبتنی بر کلید خصوصی
2. **Loopback Binding:** گت‌وی فقط روی loopback گوش می‌دهد (در تنظیمات پیش‌فرض)
3. **Plugin Allowlist:** فقط پلاگین‌های تایید شده اجازه اجرا دارند
4. **Sandboxing:** ابزارهای حساس در محیط ایزوله اجرا می‌شوند

## 🚀 نتیجه‌گیری

اگر کنترل کامل، امنیت داده، و سفارشی‌سازی عمیق می‌خواهید → **OpenClaw**

اگر راحتی استقرار و مدیریت ابری → **KiloClaw**

هر دو روی یک زیربنای مشترک (Gateway + Skills + Model Providers) ساخته شده‌اند، اما رویکرد متفاوتی به استقرار و کنترل دارند.
