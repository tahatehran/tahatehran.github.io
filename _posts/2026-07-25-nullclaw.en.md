NullClaw is a Zig-powered AI agent runtime that fits in just 678KB, making it one of the lightest full-featured AI agents available. Despite its tiny size, it supports 50+ model providers and 19 messaging channels.

## 🪶 What is NullClaw?

NullClaw is a from-scratch AI agent runtime written in Zig that:
- **Boots in under 2ms** on any platform
- **Uses less than 1MB of RAM** at idle
- **Supports 50+ LLM providers** (OpenAI, Anthropic, Google, Ollama, etc.)
- **Connects to 19 messaging channels** (WhatsApp, Telegram, Discord, Slack, etc.)
- **Includes a full skill system** with hot-reloading

## 📊 Size Comparison

| Runtime | Binary Size | RAM Usage | Startup Time |
|---------|------------|-----------|--------------|
| **NullClaw** | **678KB** | **< 1MB** | **< 2ms** |
| OpenClaw (Node.js) | ~50MB | ~100MB | ~500ms |
| NanoClaw | ~15MB | ~30MB | ~2s |
| IronClaw | ~5MB | ~5MB | ~10ms |

## 🔧 Architecture

```
NullClaw (678KB)
├── Zig Core (Memory Safe, Zero-Copy)
├── Provider Layer (50+ LLM APIs)
├── Channel Layer (19 Messaging Platforms)
├── Skill Engine (Hot-Reloadable WASM)
└── Storage (Embedded SQLite)
```

## 🚀 Getting Started

```bash
# Download NullClaw
curl -fsSL https://nullclaw.dev/install.sh | bash

# Or build from source
git clone https://github.com/nullclaw/nullclaw
cd nullclaw
zig build -Doptimize=ReleaseFast

# Run
./nullclaw --config config.toml
```

### Minimal Configuration
```toml
[provider]
type = "openai"
api_key = "sk-..."

[channels]
telegram = { token = "..." }
discord = { token = "..." }

[skills]
directory = "./skills"
hot_reload = true
```

## 📦 Skill System

NullClaw's skill system supports:
- **WASM skills:** Sandboxed execution
- **Shell skills:** Direct command execution
- **HTTP skills:** API integrations
- **Lua scripts:** Lightweight automation

## 💡 Use Cases

- **IoT Devices:** Run AI agents on Raspberry Pi, ESP32, etc.
- **Embedded Systems:** Automotive, industrial controllers
- **Edge Computing:** Low-latency local inference
- **Development:** Quick prototyping and testing
- **Production:** High-density multi-tenant deployments
