Kilo Auto Model یک قابلیت مسیریابی خودکار مدل در Kilo Code است که بهترین مدل هوش مصنوعی را برای هر وظیفه کدنویسی بدون نیاز به تغییر دستی مدل انتخاب می‌کند. اگر از دستیاران کدنویسی هوش مصنوعی استفاده می‌کنید و وقت خود را صرف انتخاب بین Claude، GPT، Gemini یا مدل‌های رایگان برای کارهای مختلف می‌کنید، Auto Model این مسیریابی را برای شما انجام می‌دهد.

## چگونه Auto Model کار می‌کند؟

به جای قفل کردن workspace به یک مدل واحد، Auto Model قبل از تصمیم به مورد استفاده، حالت فعلی و شکل وظیفه را بررسی می‌کند. وظیفه برنامه‌ریزی ممکن است به استدلال فرانتیر نیاز داشته باشد. ویرایش کد معمولی ممکن است با یک مدل پرداخت شده متعادل کافی باشد. یک توضیح سریع حتی ممکن است به یک مدل رایگان روی OpenRouter هدایت شود.

مسیر مسیریابی:

۱. **یک لایه انتخاب کنید** — Frontier، Balanced یا Free.
۲. **حالت‌ها را تغییر دهید** — Ask، Code، Debug یا Architect.
۳. **Kilo مسیریابی می‌کند** — پلتفرم بهترین مدل موجود را برای آن ترکیب انتخاب می‌کند.

## سه لایه

- **Frontier**: به جدیدترین مدل‌های پرداخت شده با استدلال قوی‌تر هدایت می‌شود. بهترین برای تصمیمات معماری، دیباگ پیچیده و طراحی سیستم.
- **Balanced**: به مدل پرداخت شده کارایی-هزینه برای کدنویسی عمومی، کاوش و پیاده‌سازی هدایت می‌شود. پیش‌فرض اگر می‌خواهید کمک‌های قوی را بدون قیمت‌های فرانتیر داشته باشید.
- **Free**: به بهترین مدل‌های رایگان OpenRouter هدایت می‌شود. مفید اگر می‌خواهید برای وظایف ساده، توضیحات یا مستندات کمک هوش مصنوعی بدون هزینه داشته باشید.

## چرا مسیریابی مدل مهم است؟

تک‌تک توسعه‌دهندگان به همه خطوط کد به مدل‌های فرانتیر نیاز ندارند. تغییر نام یک متغیر، یک تست واحد یا به‌روزرسانی README به همان مدلی نیاز ندارد که یک بازطراحی کامل معماری نیاز دارد. Auto Model مالی انتخاب مدل را حذف می‌کند و شما را در کنترل نگه می‌دارد.

همچنان می‌توانید هر وقت خواستید به صورت دستی هر مدل میزبانی‌شده، BYOK یا محلی را انتخاب کنید. Auto Model یک مسیر پیش‌فرض است، نه یک محدودیت.

## یکپارچگی با Kilo Code

Auto Model درون Kilo Code، افزونه کدنویسی هوش مصنوعی برای VS Code، IDEهای JetBrains و CLI کار می‌کند. Kilo Code دسترسی شما را به ۵۰۰+ مدل شامل Claude، GPT، Gemini، مدل‌های وزن-باز و مدل‌های محلی Ollama فراهم می‌کند. Auto Model روی آن کاتالوگ قرار می‌گیرد و به صورت هوشمند مسیریابی می‌کند.

## چه زمانی از Auto Model استفاده کنیم

- وقتی اغلب مدل را بسته به وظیفه تغییر می‌دهید.
- وقتی می‌خواهید بار شناختی را هنگام کدنویسی کاهش دهید.
- وقتی می‌خواهید کنترل هزینه را بدون فدا کردن قابلیت در وظایف سخت داشته باشید.
- وقتی از چند حالت — Ask، Code، Debug، Architect — استفاده می‌کنید و می‌خواهید مدل مناسب هر کدام.

Auto Model به خصوص مفید است اگر به Kilo Pass مشترک شده باشید، زیرا اعتبارات پاداش حتی مسیریابی فرانتیر را ارزان‌تر از حالت دیگر می‌کند.
