FastClaw is a lightweight multi-agent runtime written in Go that provides personalities, memory, skills, and sandbox isolation for running multiple AI agents simultaneously.

## ⚡ What is FastClaw?

FastClaw is a Go binary that:
- **Runs multiple agents** with distinct personalities
- **Provides shared memory** across agent teams
- **Includes a skill marketplace** for extending capabilities
- **Sandbox-isolates** each agent for security
- **Supports hot-swapping** agents without downtime

## 🏗️ Architecture

```
FastClaw Runtime (Go Binary)
├── Agent Pool
│   ├── Agent A (Personality: Analyst)
│   ├── Agent B (Personality: Writer)
│   └── Agent C (Personality: Reviewer)
├── Shared Memory (Redis/SQLite)
├── Skill Engine
└── Sandbox Layer (gVisor/Firecracker)
```

## 🔑 Key Features

### 1. Multi-Agent Orchestration
```yaml
agents:
  - name: analyst
    personality: "Data-focused, precise, analytical"
    model: gpt-4
    skills: [data-analysis, visualization]
  
  - name: writer
    personality: "Creative, clear, engaging"
    model: claude-3
    skills: [content-writing, editing]
  
  - name: reviewer
    personality: "Critical, detail-oriented, thorough"
    model: gpt-4
    skills: [code-review, testing]
```

### 2. Shared Memory
```json
{
  "memory": {
    "backend": "redis",
    "namespace": "fastclaw-team",
    "ttl": "7d",
    "sharding": "consistent-hash"
  }
}
```

### 3. Sandbox Isolation
Each agent runs in its own sandbox:
```yaml
sandbox:
  type: gvisor
  network: isolated
  filesystem: read-only
  memory_limit: 256MB
  cpu_limit: 0.5
```

## 🚀 Getting Started

```bash
# Install FastClaw
go install github.com/fastclaw-ai/fastclaw@latest

# Or download binary
curl -fsSL https://fastclaw.dev/install.sh | bash

# Run with default config
fastclaw start --config agents.yaml

# Check agent status
fastclaw status
```
